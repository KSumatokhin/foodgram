**Проект «Фудграм»**

Здравствуйте, подскажите по фильтрации тегов для рецептов.


Задача у нас такая, что при запросе:
`GET http://127.0.0.1:8000/api/recipes/?tags=lunch`
Должен получиться список рецептов для `lunch` - получаю список 3 рецепта

При:
`GET http://127.0.0.1:8000/api/recipes/?tags=dinner`
Должен получиться список рецептов для `dinner`  - получаю список 1 рецепт

А для совместного запроса:
`GET http://127.0.0.1:8000/api/recipes/?tags=lunch&tags=dinner`
Должен получиться список рецептов, который будет содержать и `lunch`, и `dinner`.

А у меня вместо 4-х рецептов получается список только для `dinner`!

Подключенный класс фильтрации:
```
class RecipeFilter(django_filters.FilterSet):
	...
    tags = django_filters.CharFilter(
        field_name='tags',
        lookup_expr='slug__exact'
    )
	...
	class Meta:
        model = Recipe
        fields = ('author', 'tags', 'is_in_shopping_cart')
	...
```

Значит формируется такой queryset:
Для 1 запроса:
`Recipe.objects.filter(tags__slug__exact='lunch')`
`Recipe.objects.filter(tags__slug__exact='dinner')`

И, как я понял, при `?tags=lunch&tags=dinner` в параметры попадает только `dinner`.

Подскажите, как это можно исправить, в какую сторону копать?
from django.db.models import Q
Recipe.objects.filter(Q(tags__slug__exact='lunch') | Q(tags__slug__exact='dinner'))
